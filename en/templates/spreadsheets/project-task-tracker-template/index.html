Design a Practical Project Task Tracker Spreadsheet

Track tasks, owners, due dates, dependencies, status, blockers, and next actions in one maintainable table.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. Use actionable task sizes
  2. Recommended columns
  3. Status definitions
  4. Weekly maintenance

Use actionable task sizes

A large label such as Build website does not show what is actually moving. Break work into units with a clear deliverable or next action.

Define who will do what and by when, rather than relying only on status.

Recommended columns

  • Task ID, project, and task name.
  • Owner, reviewer, and priority.
  • Start date, due date, and completion date.
  • Status, progress, and next action.
  • Dependency, blocker, and last update.

Status definitions

  • Not started: ready but not begun.
  • In progress: currently being worked.
  • Waiting for review: identify the reviewer.
  • On hold: include reason and restart condition.
  • Done: completion criteria met and verified.

Weekly maintenance

  1. Filter overdue and due-this-week items.
  2. Assign each blocker to an owner.
  3. Rewrite tasks with unclear completion criteria.
  4. Close unnecessary work with a reason.
  5. Record one next action before the next review.
Key point

A clear next action and blocker is usually more useful than a subjective progress percentage.
