How to Use Excel VLOOKUP: Exact Match and Column Index Risks

Understand VLOOKUP syntax, exact match, lookup columns, column indexes, #N/A, and type mismatches.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. Fundamentals
  2. Syntax
  3. Steps
  4. Business examples
  5. Common errors

Fundamentals

VLOOKUP searches the leftmost column of a table and returns a value from a specified column in the same row. Unspecified match mode and fixed column indexes are common error sources.

Syntax

=VLOOKUP(A2,$F$2:$H$100,3,FALSE)

Steps

  1. Confirm the lookup value is in the leftmost table column.
  2. Limit the lookup range to required columns.
  3. Count the return column from the left.
  4. Use FALSE for exact match in ordinary ID lookups.
  5. Review unmatched rows and duplicate keys separately.

Business examples

  • returning product name or price from a code
  • returning department from employee ID
  • displaying account owner from customer ID
  • maintaining lookup logic in older workbook environments

Common errors

  • lookup column is not leftmost
  • FALSE is omitted for exact matching
  • inserted columns change the fixed index
  • spaces or number/text mismatches cause #N/A
Key point

After building the formula, test zero values, blanks, duplicates, and boundary cases.

Frequently asked questions

Where do I enter the formula?

Select the result cell and enter the formula beginning with an equals sign. Then verify the highlighted references and the formula bar.

What if the result does not update?

Check automatic calculation, whether referenced cells are numeric, and whether external links or filters affect the result.

Can I copy the formula to other rows?

Yes, but verify that relative and absolute references behave as intended before filling the formula down.
