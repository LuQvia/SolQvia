Join Two Tables by a Common Key with Excel Power Query

Add master-data attributes to transactions without filling the workbook with large lookup formulas.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. How merge works
  2. Merge workflow
  3. Common join types
  4. Reconciliation

How merge works

A Power Query merge matches two queries using one or more common columns and adds related fields. A typical example is adding product category from a product master to sales transactions.

The key columns must have compatible data types and consistent spacing and characters.

Merge workflow

  1. Convert both ranges to Excel tables and load them into Power Query.
  2. Open the primary transaction query.
  3. Choose Merge Queries and select the key columns in both tables.
  4. Choose the required join type.
  5. Expand only the related columns you need.

Common join types

  • Left outer: keep every primary row and add available master data.
  • Inner: keep only records that match in both tables.
  • Left anti: identify transaction keys missing from the master.
  • Full outer: review all matches and mismatches.

Reconciliation

  • Confirm that the primary row count did not grow unexpectedly.
  • Check that the master key is unique.
  • Extract unmatched IDs for correction.
  • Investigate null values after expansion.
Key point

Duplicate keys in the master can multiply transaction rows, so validate uniqueness before merging.
