Remove Duplicate Rows in Excel Without Deleting Valid Records

Define the duplicate key, review candidates, back up the data, and reconcile totals before permanent deletion.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. Define duplicate correctly
  2. Safer workflow
  3. Do not delete when
  4. Prevent recurrence

Define duplicate correctly

Excel Remove Duplicates deletes rows based on the selected column values. Two people may share a name, so use a business key such as customer ID or a defined combination of fields.

The first occurrence remains, so decide which record should be kept before running the command.

Safer workflow

  1. Copy the source to another sheet or workbook.
  2. Highlight duplicate candidates with conditional formatting.
  3. Define the priority rule for the record to keep.
  4. Confirm the range and key columns before removing duplicates.
  5. Reconcile row counts, amounts, and ID lists before and after.

Do not delete when

  • Repeated customer rows represent valid transactions.
  • The displayed value is the same but the records are different.
  • Rows represent change history or versions.
  • There is no clear rule for selecting the surviving record.

Prevent recurrence

  • Assign a unique ID at entry.
  • Separate master records from transaction history.
  • Run a recurring duplicate-candidate report.
  • Normalise spaces, character width, and letter case.
Key point

Temporarily filtering duplicates is different from permanently deleting records.
