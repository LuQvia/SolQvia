Excel Conditional Formatting Basics for Duplicates, Deadlines, and Exceptions

Highlight duplicates, overdue items, missing inputs, and exceptions while avoiding rule conflicts and range errors.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. Fundamentals
  2. Syntax
  3. Steps
  4. Business examples
  5. Common errors

Fundamentals

Conditional formatting changes appearance based on values or formulas. Color is a visual aid, not a substitute for status data or auditable fields.

Syntax

期限超過の数式例
=AND($D2<TODAY(),$E2<>"完了")
重複確認
=COUNTIF($A:$A,$A2)>1

Steps

  1. Define the exception in plain language.
  2. Build the formula relative to the first applied row.
  3. Check absolute column and relative row references.
  4. Review rule priority and stop conditions.
  5. Test blanks, boundary dates, and completed records.

Business examples

  • highlighting duplicate employee or customer IDs
  • highlighting overdue incomplete rows
  • marking missing required cells
  • flagging values far from a target

Common errors

  • formula row does not match the first applied row
  • status is stored only as color
  • overlapping rules create unexpected formatting
  • excessive whole-column formulas reduce performance
Key point

After building the formula, test zero values, blanks, duplicates, and boundary cases.

Frequently asked questions

Where do I enter the formula?

Select the result cell and enter the formula beginning with an equals sign. Then verify the highlighted references and the formula bar.

What if the result does not update?

Check automatic calculation, whether referenced cells are numeric, and whether external links or filters affect the result.

Can I copy the formula to other rows?

Yes, but verify that relative and absolute references behave as intended before filling the formula down.
