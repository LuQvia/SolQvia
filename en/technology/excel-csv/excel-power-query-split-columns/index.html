Split Address, Name, and Code Columns with Power Query

Turn delimited or fixed-width text into reusable columns and preserve the transformation for future refreshes.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. When Power Query is better than manual editing
  2. Split methods
  3. Workflow
  4. Validation

When Power Query is better than manual editing

Recurring formats such as Last,First, region-city, or product_branch can be split with a saved query step that runs again on refresh.

Duplicate the original column first when you need to preserve the raw value.

Split methods

  • Delimiter such as comma, space, or hyphen.
  • Fixed number of characters.
  • Specific character position.
  • Transition from lowercase to uppercase or digit to non-digit.

Workflow

  1. Set the target column to text.
  2. Duplicate it when the raw value must remain.
  3. Choose the split rule and occurrence.
  4. Rename the resulting columns and set their data types.
  5. Review exceptions and unexpected column counts.

Validation

  • Check whether the delimiter also appears inside the content.
  • Normalise full-width and half-width spaces when relevant.
  • Do not force variable addresses into an unsafe fixed structure.
  • Reconcile results before deleting the original field.
Key point

Collect exceptions before finalising the rule to reduce incorrect transformations.
