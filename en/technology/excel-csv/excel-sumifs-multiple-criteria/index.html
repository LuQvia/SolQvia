How to Use Excel SUMIFS with Multiple Criteria

Combine store, owner, and date criteria to summarize transactions with a formula that copies cleanly across a report.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. SUMIFS basics
  2. Store and month example
  3. References that copy correctly
  4. Prevent calculation errors

SUMIFS basics

SUMIFS adds values from rows that meet all specified criteria. It is widely used to summarize transactions by store, owner, period, status, or product.

The syntax is SUMIFS(sum_range, criteria_range1, criteria1, ...). Note that SUMIF uses a different argument order.

Store and month example

This assumes A contains dates, B stores, D sales, H3 the store name, and I2 the first day of the selected month.

=SUMIFS($D$2:$D$1000,$B$2:$B$1000,$H3,$A$2:$A$1000,">="&I$2,$A$2:$A$1000,"<"&EDATE(I$2,1))

References that copy correctly

  • Lock source ranges with absolute references.
  • For criteria that change down rows, lock only the column.
  • For month headers that change across columns, lock only the row.
  • Concatenate comparison operators and cell references with &.

Prevent calculation errors

  • Keep the sum range and every criteria range the same size.
  • Confirm dates are stored as Excel dates, not text.
  • Remove unintended spaces from criteria labels.
  • Use COUNTIFS when you need to distinguish no matching rows from a legitimate zero total.
Key point

Clean source data is more important than making the summary formula increasingly complex.
