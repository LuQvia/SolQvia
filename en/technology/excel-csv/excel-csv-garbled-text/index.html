Why CSV Text Looks Garbled in Excel and How to Fix It

Understand encoding differences and import CSV files through Excel's data tools instead of opening them blindly.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. Why garbled text occurs
  2. Recommended import method
  3. Prevent problems when exporting
  4. Common mistakes

Why garbled text occurs

CSV defines a delimited table format, not one universal text encoding. A file may be UTF-8, while Excel interprets it with another encoding, producing unreadable characters.

Importing through Excel's data tools lets you select the encoding and delimiter before the values enter the worksheet.

Recommended import method

  1. Open a blank Excel workbook.
  2. Choose Data and then From Text/CSV.
  3. Select the CSV file.
  4. In the preview, choose UTF-8 or the appropriate file origin.
  5. Confirm the delimiter and column interpretation before loading.

Prevent problems when exporting

  • UTF-8 with a byte-order mark may be recognized more reliably by Excel.
  • Confirm the encoding required by older business systems.
  • Import postal codes and identifiers as text when leading zeros matter.
  • Review automatic date and large-number conversion in the preview.

Common mistakes

  • Saving over the source after opening it with the wrong encoding.
  • Changing only the filename extension to .xlsx.
  • Assuming every CSV uses the same legacy encoding.
  • Loading phone numbers or IDs as numeric values and losing leading zeros or precision.
Key point

CSV imports can also corrupt dates and long identifiers. Check data types, not only visible character encoding.
