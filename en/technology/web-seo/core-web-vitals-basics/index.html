Core Web Vitals: Practical Design Basics

Understand LCP, INP, and CLS and avoid performance regressions when adding images, scripts, analytics, and ads.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. The three metrics
  2. Common good thresholds
  3. Improvement priority
  4. When adding ads

The three metrics

Core Web Vitals measure loading performance, interaction responsiveness, and visual stability. The main metrics are LCP, INP, and CLS.

Treat them as user-experience indicators, not only search metrics. Readers must be able to load, read, and use the page reliably.

Common good thresholds

  • LCP: loading of the largest main content element; good at 2.5 seconds or less.
  • INP: responsiveness to interaction; good at 200 milliseconds or less.
  • CLS: unexpected layout movement; good at 0.1 or less.

Improvement priority

  1. Optimize the largest above-the-fold image and serve an appropriate size and format.
  2. Reduce unnecessary JavaScript and third-party tags.
  3. Reserve dimensions or aspect ratios for images, ads, and embeds.
  4. Limit web fonts or use efficient subsets.
  5. Compare real-user field data with laboratory diagnostics from PageSpeed Insights.

When adding ads

  • Reserve ad space instead of inserting it after layout.
  • Avoid excessive advertising before the main content.
  • Prevent duplicate consent, analytics, and advertising scripts.
  • Compare Core Web Vitals and total revenue before and after implementation.
Key point

Optimize for task completion and reading quality, not a score in isolation.
