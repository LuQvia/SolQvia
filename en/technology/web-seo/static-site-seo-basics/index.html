Technical SEO Basics Before Publishing a Static Website

Check page titles, descriptions, canonicals, sitemaps, structured data, robots rules, and internal links before launch.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. Page-level metadata
  2. Minimum checklist
  3. Article structured data
  4. After launch

Page-level metadata

A static site can provide a unique title, description, canonical URL, and heading for every page. Reusable templates should still receive page-specific content aligned with search intent.

Canonical URLs should use the final HTTPS domain. Remove staging domains and default hosting URLs before launch.

Minimum checklist

  • One primary h1 accurately represents the page.
  • Title and description are unique.
  • The canonical is self-referencing and uses the final HTTPS domain.
  • robots.txt does not accidentally block important pages.
  • sitemap.xml contains only public canonical pages.
  • The 404 page returns an appropriate not-found response.
  • Internal links work with the deployed path structure.

Article structured data

Article or BlogPosting JSON-LD can describe article pages. Keep headline, datePublished, dateModified, author, publisher, and mainEntityOfPage consistent with visible content. Structured data does not guarantee a particular search appearance.

After launch

  1. Verify the domain in Search Console and submit the sitemap.
  2. Inspect the home page and important articles.
  3. Review crawl errors, duplicates, and excluded-page reasons.
  4. When content changes, update both visible dates and sitemap metadata accurately.
This article provides general information. Check official sources for current specifications, prices, rules, and opening hours.
