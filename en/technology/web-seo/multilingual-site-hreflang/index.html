How to Implement hreflang on a Multilingual Website

Connect Japanese and English URLs with reciprocal hreflang annotations and self-referencing canonicals.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. What hreflang communicates
  2. URL structure
  3. HTML example
  4. Implementation checklist

What hreflang communicates

hreflang tells search engines that pages contain equivalent or closely related content for different languages or regions. Each language URL should reference the complete set of alternates.

It is not a direct ranking boost. It helps search engines show the most appropriate language version.

URL structure

Separate URLs for each language are easier to crawl, link, and share than changing the language of one URL dynamically.

https://example.com/ja/article/
https://example.com/en/article/

HTML example

Include the same reciprocal set on both the Japanese and English pages. One-way annotations may be ignored.

<link rel="alternate" hreflang="ja" href="https://example.com/ja/article/">
<link rel="alternate" hreflang="en" href="https://example.com/en/article/">
<link rel="alternate" hreflang="x-default" href="https://example.com/en/article/">

Implementation checklist

  • Every page includes a self-reference.
  • Every alternate page links back.
  • The canonical points to the current language URL, not a different language.
  • Do not list missing URLs or unnecessary redirects.
  • Use valid language and, when needed, region codes.
  • Keep sitemap and HTML annotations consistent.
Key point

Localize titles, headings, navigation, calls to action, and supporting text—not only the main article body.
