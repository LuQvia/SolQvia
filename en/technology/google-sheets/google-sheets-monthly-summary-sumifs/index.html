How to Build a Monthly Summary in Google Sheets with SUMIFS

Summarize sales by month, store, or owner using date boundaries and multiple SUMIFS criteria.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. Prepare the source table
  2. Core formula
  3. Expand the summary
  4. Common errors

Prepare the source table

Keep one transaction per row. A simple structure might use column A for date, B for store, C for owner, and D for sales. For monthly totals, use two date conditions: on or after the first day of the month, and before the first day of the next month.

Core formula

This example assumes H2 contains the first day of the target month and H3 contains the store name. The exclusive upper boundary also works well when source dates include times.

=SUMIFS($D:$D,$A:$A,">="&$H$2,$A:$A,"<"&EDATE($H$2,1),$B:$B,H$3)

Expand the summary

  1. List stores or owners down the rows.
  2. Place the first day of each month across the columns.
  3. Keep source ranges absolute and criteria cells relative where appropriate.
  4. Verify that labels match the master list exactly.
  5. Prevent invalid dates or inconsistent labels at data-entry time.

Common errors

  • Dates are stored as text instead of date values.
  • Names differ because of spaces or character variations.
  • The sum range and criteria ranges have different sizes.
  • A fixed month-end date fails for February or leap years.
  • Subtotal rows are mixed into the source data and counted twice.
Key point

Keep source data free of subtotal rows. Build all subtotals and totals in the reporting layer.
