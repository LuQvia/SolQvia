Google Sheets QUERY Function: A Practical Introduction

Use SELECT, WHERE, and GROUP BY to turn transaction rows into a summarized table in one formula.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. When QUERY is useful
  2. Sales by store
  3. Use a month selected in a cell
  4. Choose the right approach

When QUERY is useful

QUERY can select columns, filter rows, and aggregate groups from a tabular range. It may replace a large grid of SUMIFS formulas when you need a dynamic summary table.

Because some formulas depend on column positions, keep the source schema stable and document what the formula does.

Sales by store

This assumes column A contains dates, B stores, and D sales. The final 1 tells QUERY that the source has one header row.

=QUERY(A:D,"select B, sum(D) where A is not null group by B label sum(D) 'Total sales'",1)

Use a month selected in a cell

Dates inside a QUERY string use the date 'yyyy-mm-dd' form. Pay close attention to quotes when concatenating cell values.

=QUERY(A:D,"select B, sum(D) where A >= date '"&TEXT(H2,"yyyy-mm-dd")&"' and A < date '"&TEXT(EDATE(H2,1),"yyyy-mm-dd")&"' group by B",1)

Choose the right approach

  • Return one metric to a known cell: SUMIFS is often clearer.
  • Return a dynamic summary table: QUERY is convenient.
  • Inconsistent labels: fix master data and validation first.
  • Very large or complex transformations: consider BigQuery or a database.
Key point

Do not type values into the cells where a QUERY result needs to expand.
