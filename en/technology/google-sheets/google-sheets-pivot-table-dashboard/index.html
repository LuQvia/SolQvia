Build a Monthly Reporting Dashboard with Google Sheets Pivot Tables

Summarise transaction-level data by month, location, owner, category, amount, and record count.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. Source-data requirements
  2. Monthly sales workflow
  3. Dashboard layout
  4. Common discrepancies

Source-data requirements

Pivot tables work best with long-form data where one row represents one record and every column has one header. Remove merged cells, subtotal rows, and multi-level headings first.

Include the date, location, owner, category, status, amount, and other fields needed for analysis in each record.

Monthly sales workflow

  1. Select the transaction range and insert a pivot table.
  2. Add location or owner to Rows.
  3. Add the date to Columns and group it by month when appropriate.
  4. Add the sales amount to Values and summarise by SUM.
  5. Add category or status fields as filters.

Dashboard layout

  • Show both revenue and record count.
  • Allow views by location, owner, and product category.
  • Confirm that new source rows are included.
  • Limit charts to the few views that support decisions.

Common discrepancies

  • Amounts stored as text cannot be summed correctly.
  • Text dates do not group by month.
  • Spelling differences split one location into multiple labels.
  • The source range excludes newly added rows.
  • Cancelled and test records are not excluded.
Key point

Standardise data types and master labels before building the pivot table.
