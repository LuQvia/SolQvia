Use doGet and doPost Correctly in Apps Script Web Apps

Build Apps Script web pages and endpoints that handle query parameters, form submissions, and JSON POST requests.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. Roles of doGet and doPost
  2. Request data
  3. JSON endpoint example
  4. Pre-deployment checklist

Roles of doGet and doPost

Apps Script runs doGet for GET requests and doPost for POST requests. Return HtmlOutput for a user interface and TextOutput for JSON or plain-text responses.

Before deployment, understand the difference between test and production URLs, the execution identity, and the selected access scope.

Request data

  • e.parameter: query or form values as single values.
  • e.parameters: repeated parameters as arrays.
  • e.postData.contents: raw POST body such as JSON.
  • e.pathInfo: additional path information after the deployment URL.

JSON endpoint example

function doPost(e) {
  try {
    const payload = JSON.parse(e.postData.contents || '{}');
    if (!payload.id) throw new Error('id is required');

    const result = { ok: true, receivedId: String(payload.id) };
    return ContentService.createTextOutput(JSON.stringify(result))
      .setMimeType(ContentService.MimeType.JSON);
  } catch (error) {
    return ContentService.createTextOutput(JSON.stringify({
      ok: false,
      error: error.message
    })).setMimeType(ContentService.MimeType.JSON);
  }
}

Pre-deployment checklist

  1. Choose whose identity the app runs as.
  2. Limit the audience to the smallest required group.
  3. Validate required fields, types, and lengths.
  4. Do not place sensitive tokens in query strings without careful design.
  5. Use duplicate prevention and locking around shared updates.
Key point

Apps Script web apps are useful for lightweight integrations. Compare other platforms when authentication, audit, or availability requirements are high.
