Build a Mail Queue Around Apps Script Sending Quotas

Track pending, sent, and failed messages, then process a controlled batch instead of sending every email in one run.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. Design around quotas
  2. Queue columns
  3. Batch example
  4. Operational controls

Design around quotas

Apps Script and Google services have quotas that can vary by account type and may change. Do not place every message in a single execution; inspect the available quota and process a bounded batch.

Even legitimate internal notifications and application results need recipient deduplication, retry control, and an audit trail.

Queue columns

  • message_id: unique send identifier.
  • to, subject, body: message content.
  • status: pending, sent, or failed.
  • attempts: number of tries.
  • next_attempt_at: scheduled retry time.
  • sent_at and last_error: delivery history.

Batch example

function processMailQueue() {
  const remaining = MailApp.getRemainingDailyQuota();
  if (remaining <= 0) return;

  const sheet = SpreadsheetApp.getActive().getSheetByName('MailQueue');
  const values = sheet.getDataRange().getValues();
  let sent = 0;

  for (let i = 1; i < values.length && sent < Math.min(remaining, 50); i++) {
    if (values[i][4] !== 'pending') continue;
    MailApp.sendEmail(values[i][1], values[i][2], values[i][3]);
    sheet.getRange(i + 1, 5, 1, 2).setValues([['sent', new Date()]]);
    sent++;
  }
}

Operational controls

  1. Deduplicate recipients and assign one ID per message.
  2. Check remaining quota before sending.
  3. Limit each run and continue with a time trigger.
  4. Retry only temporary failures; hold permanent errors for review.
  5. Restrict access to recipient lists and message content.
Key point

A send may succeed before the status update fails, so the design needs an identifier that can detect duplicate delivery.
