Speed Up Apps Script with Batch Reads and Writes

Reduce Apps Script execution time by reading ranges into arrays, processing them in memory, and writing results back in one operation.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. Why scripts become slow
  2. Batch-processing workflow
  3. Example
  4. Design checks

Why scripts become slow

Calls to services such as SpreadsheetApp are usually more expensive than calculations inside JavaScript. A loop that repeatedly calls getValue and setValue can become slow as the row count grows.

The standard pattern is to read the required range once with getValues, transform the two-dimensional array in memory, and write the final array with setValues.

Batch-processing workflow

  1. Determine the exact start row, start column, row count, and column count.
  2. Read the range into a two-dimensional array with getValues().
  3. Transform the array with map or a loop.
  4. Confirm that the output dimensions match the destination range.
  5. Write the entire result with one setValues() call.

Example

function updateStatuses() {
  const sheet = SpreadsheetApp.getActive().getSheetByName('Tasks');
  const lastRow = sheet.getLastRow();
  if (lastRow < 2) return;

  const values = sheet.getRange(2, 1, lastRow - 1, 4).getValues();
  const output = values.map(row => {
    const due = row[2];
    const done = row[3] === true;
    const status = done ? 'Done' : (due instanceof Date && due < new Date() ? 'Overdue' : 'In progress');
    return [status];
  });

  sheet.getRange(2, 5, output.length, 1).setValues(output);
}

Design checks

  • Read only the actual data range instead of entire columns when possible.
  • Combine output operations to reduce service calls.
  • Preserve dates, booleans, and numbers instead of converting everything to text.
  • For very large jobs, process chunks and save a checkpoint.
Key point

Count service calls when diagnosing performance. Fewer calls usually produce the largest improvement.
