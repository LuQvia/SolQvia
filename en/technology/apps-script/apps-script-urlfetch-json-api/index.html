Call JSON APIs from Apps Script with UrlFetchApp

Handle GET and POST requests, headers, JSON payloads, response codes, and safe API configuration.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. External API workflow
  2. POST example
  3. Specification checklist
  4. Operational controls

External API workflow

UrlFetchApp sends HTTP requests from Apps Script to external services. Configure the URL, method, headers, body, and error handling according to the API specification.

Keep API keys and tokens out of source code and avoid writing full credentials to logs.

POST example

function createItem(payload) {
  const token = PropertiesService.getScriptProperties().getProperty('API_TOKEN');
  if (!token) throw new Error('API_TOKEN is not configured');

  const response = UrlFetchApp.fetch('https://api.example.com/items', {
    method: 'post',
    contentType: 'application/json',
    headers: { Authorization: `Bearer ${token}` },
    payload: JSON.stringify(payload),
    muteHttpExceptions: true
  });

  const status = response.getResponseCode();
  const body = response.getContentText();
  if (status < 200 || status >= 300) {
    throw new Error(`API error ${status}: ${body.slice(0, 300)}`);
  }
  return JSON.parse(body);
}

Specification checklist

  • Authentication method and token lifetime.
  • Content-Type and character encoding.
  • Success and error status codes.
  • Pagination format.
  • Rate limits and retry rules.
  • Duplicate behavior when a request is sent again.

Operational controls

  1. Test with a small non-production record first.
  2. Store only the response fields you need.
  3. Retry only rate limits and temporary server failures, with a limit.
  4. Split large volumes into queued trigger runs.
  5. Record the API version and last verification date.
Key point

When muteHttpExceptions is enabled, the code must still inspect the response status.
