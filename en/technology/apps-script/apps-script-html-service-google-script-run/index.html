Build an Apps Script Form with HTML Service and google.script.run

Separate the browser interface from server-side validation and spreadsheet updates in an Apps Script web app.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. Separate UI and server logic
  2. Minimum server setup
  3. Client-side call
  4. Quality checks

Separate UI and server logic

HTML Service can render HTML, CSS, and browser-side JavaScript. The page can call server-side Apps Script functions asynchronously with google.script.run.

Keep input and display logic in the browser, while server-side code handles authorization, data access, and validation. Never trust browser-side validation alone.

Minimum server setup

// Code.gs
function doGet() {
  return HtmlService.createHtmlOutputFromFile('Index')
    .setTitle('Request Form');
}

function saveRequest(data) {
  if (!data || !String(data.name || '').trim()) {
    throw new Error('Name is required');
  }
  const sheet = SpreadsheetApp.getActive().getSheetByName('Requests');
  sheet.appendRow([new Date(), String(data.name).trim(), String(data.note || '')]);
  return { ok: true };
}

Client-side call

document.querySelector('#form').addEventListener('submit', event => {
  event.preventDefault();
  const button = event.submitter;
  button.disabled = true;

  google.script.run
    .withSuccessHandler(() => {
      button.disabled = false;
      document.querySelector('#message').textContent = 'Saved';
    })
    .withFailureHandler(error => {
      button.disabled = false;
      document.querySelector('#message').textContent = error.message;
    })
    .saveRequest({ name: form.name.value, note: form.note.value });
});

Quality checks

  • Disable submission while the request is running.
  • Always provide success and failure handlers.
  • Do not embed secrets or access tokens in HTML.
  • Load active external content over HTTPS.
  • Test labels, keyboard use, and understandable errors.
Key point

HTML Service runs in an iframe sandbox, so some behaviors differ from a normal standalone web page.
