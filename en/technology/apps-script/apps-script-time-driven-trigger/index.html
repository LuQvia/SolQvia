How to Automate Recurring Tasks with Apps Script Time-driven Triggers

Schedule daily, hourly, or weekly Apps Script jobs and design them to avoid duplicate or overlapping execution.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. What time-driven triggers do
  2. Create a trigger in the interface
  3. Create a trigger in code
  4. Reliability patterns

What time-driven triggers do

Time-driven triggers run an Apps Script function automatically on a schedule. Common uses include daily summaries, queue processing, recurring notifications, and backups.

Execution may occur within the selected time window rather than at an exact second. Design the workflow so it does not depend on precise real-time scheduling.

Create a trigger in the interface

  1. Open Triggers in the Apps Script sidebar.
  2. Add a trigger and select the function to run.
  3. Choose Time-driven as the event source.
  4. Select a minute, hourly, daily, or weekly interval.
  5. Review failure notifications and save the trigger.

Create a trigger in code

function createDailyTrigger() {
  deleteTriggers_('dailyJob');
  ScriptApp.newTrigger('dailyJob')
    .timeBased()
    .everyDays(1)
    .atHour(9)
    .create();
}

function deleteTriggers_(handlerName) {
  ScriptApp.getProjectTriggers()
    .filter(t => t.getHandlerFunction() === handlerName)
    .forEach(t => ScriptApp.deleteTrigger(t));
}

function dailyJob() {
  // Daily processing goes here.
}

Reliability patterns

  • Remove an existing trigger before creating another one with the same purpose.
  • Store processed IDs or a checkpoint so reruns do not duplicate work.
  • Use LockService to reduce overlapping execution.
  • Log failures from external APIs and email delivery.
  • Split long jobs and save the next position in properties.
Key point

Aim for idempotent jobs: running the same job twice should not corrupt or duplicate the result.
