How to Generate PDFs from Google Docs with Apps Script

Copy a Google Docs template, replace placeholders, and save the result as a PDF in Google Drive.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. The document-generation flow
  2. What you need
  3. Sample code
  4. Quality checks

The document-generation flow

Apps Script can copy a Google Docs template, replace placeholders, and convert the completed document to PDF. This pattern works well for estimates, confirmations, certificates, and other documents with a fixed layout and variable data.

Copy the template for each run instead of editing the master document. This protects the original layout and makes failures easier to recover from.

What you need

  • The Google Docs template ID
  • The destination Drive folder ID
  • A mapping of placeholders to values
  • A consistent file-naming rule

Sample code

function createPdf(data) {
  const templateId = 'DOCUMENT_TEMPLATE_ID';
  const folder = DriveApp.getFolderById('OUTPUT_FOLDER_ID');
  const copy = DriveApp.getFileById(templateId)
    .makeCopy(`document_${data.id}`, folder);

  const doc = DocumentApp.openById(copy.getId());
  const body = doc.getBody();
  body.replaceText('{{NAME}}', data.name);
  body.replaceText('{{DATE}}', data.date);
  body.replaceText('{{AMOUNT}}', String(data.amount));
  doc.saveAndClose();

  const pdfBlob = copy.getAs(MimeType.PDF)
    .setName(`document_${data.id}.pdf`);
  const pdfFile = folder.createFile(pdfBlob);
  copy.setTrashed(true);
  return pdfFile.getUrl();
}

Quality checks

  • Avoid placeholder patterns that unintentionally behave as regular expressions.
  • Test long names and addresses for layout overflow.
  • Format dates and amounts consistently before replacement.
  • Confirm PDF creation before deleting the temporary document copy.
  • For regulated documents, separately verify required fields and retention rules.
Key point

For batch jobs, process a limited number of records per run and record completion status so the job can resume safely.
