How to Send Google Forms Responses to Slack

Connect a Google Forms submit trigger to a Slack Incoming Webhook with Apps Script and secure the webhook URL in script properties.

Published: 2026-07-22Last updated: 2026-07-22
In this guide
  1. Required components
  2. Setup steps
  3. Notification code
  4. Operational safeguards

Required components

The workflow combines a Slack Incoming Webhook with a Google Forms submit trigger. Apps Script posts JSON to the webhook by using UrlFetchApp.fetch().

Treat the webhook URL as a secret. Instead of placing it in a shared spreadsheet cell or public repository, store it in Apps Script properties.

Setup steps

  1. Enable an Incoming Webhook in Slack and obtain the URL for the target channel.
  2. Add SLACK_WEBHOOK_URL to Apps Script project properties.
  3. Create a function that receives the form-submit event.
  4. Add an installable trigger and submit a test response.

Notification code

function onFormSubmit(e) {
  const url = PropertiesService.getScriptProperties()
    .getProperty('SLACK_WEBHOOK_URL');
  if (!url) throw new Error('Webhook URL is not configured.');

  const text = Object.entries(e.namedValues)
    .map(([key, value]) => `*${key}*\n${value.join(', ')}`)
    .join('\n\n');

  const response = UrlFetchApp.fetch(url, {
    method: 'post',
    contentType: 'application/json',
    payload: JSON.stringify({ text }),
    muteHttpExceptions: true
  });

  if (response.getResponseCode() >= 300) {
    throw new Error(response.getContentText());
  }
}

Operational safeguards

  • Do not store the webhook URL in a broadly shared sheet.
  • Log and inspect the HTTP response when delivery fails.
  • Confirm channel membership and retention rules before posting personal data.
  • For quiet hours, queue messages and deliver them later with a time-driven trigger.
Key point

Start with a simple text payload. Move to Block Kit only after the basic delivery flow is stable.
