多言語サイトでhreflangを正しく設定する方法

日本語・英語ページの対応関係をGoogleへ伝えるURL設計と相互指定の基本を解説します。

公開日: 2026-07-22最終更新: 2026-07-22

  2. URL設計

hreflangの役割

hreflangは、同じまたは近い内容のページについて、言語や地域別の対応関係を検索エンジンへ伝える仕組みです。日本語版と英語版を別URLで公開する場合、それぞれのページから両方のURLを指定します。

hreflangは検索順位を直接上げる設定ではありません。適切な言語版を検索結果へ表示しやすくするための補助情報です。

URL設計

同じURLでブラウザ言語に応じて本文を切り替えるより、言語ごとに固有URLを持たせる方がクロールと共有が明確になります。

https://example.com/ja/article/
https://example.com/en/article/

HTMLの例

日本語ページと英語ページの両方に同じ対応セットを記述します。片方向だけの指定は無視される可能性があります。

<link rel="alternate" hreflang="ja" href="https://example.com/ja/article/">
<link rel="alternate" hreflang="en" href="https://example.com/en/article/">
<link rel="alternate" hreflang="x-default" href="https://example.com/en/article/">

実装チェック

  • 各言語ページが自己参照を含む。
  • 対応先ページからも戻りの指定がある。
  • canonicalが別言語ページではなく、そのページ自身を指す。
  • 存在しないURLやリダイレクト先を指定しない。
  • 言語コードと必要な場合の地域コードを正しく使う。
  • サイトマップで指定する場合もHTMLと矛盾させない。
ポイント

翻訳ページのタイトル、見出し、ナビゲーション、CTAも対象言語へ統一してください。本文だけ翻訳した混在ページはユーザー体験を損ないます。

掲載内容は一般的な情報です。実際の仕様、料金、制度、営業時間などは公式情報をご確認ください。
