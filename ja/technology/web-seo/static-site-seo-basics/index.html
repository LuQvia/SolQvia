静的サイト公開前に確認するSEO設定

title、description、canonical、サイトマップ、構造化データなど最低限の技術設定を整理します。

公開日: 2026-07-22最終更新: 2026-07-22
この記事の内容
  1. ページ単位の設定
  2. 最低限のチェック
  3. 記事構造化データ

ページ単位の設定

静的サイトでも、各ページに固有のtitle、description、canonical、見出しを設定できます。同じテンプレートを使っていても、記事ごとの検索意図に合わせて内容を変えます。

canonicalは公開する正規URLを絶対URLで指定し、テスト用ドメインやGitHub PagesのURLを残さないようにします。

最低限のチェック

  • 1ページに主要なh1が1つあり、内容を正確に表す。
  • titleとdescriptionがページごとに固有。
  • canonicalが自己参照し、HTTPSの正規ドメインを使う。
  • robots.txtが重要ページを誤って拒否していない。
  • sitemap.xmlに公開ページだけが含まれる。
  • 404ページが適切なHTTPステータスで返る。
  • 内部リンクが相対・絶対URLとも正しく動く。

記事構造化データ

記事ページではArticleまたはBlogPostingのJSON-LDを追加できます。headline、datePublished、dateModified、author、publisher、mainEntityOfPageなどを実際の表示内容と一致させます。構造化データは検索結果の表示を保証するものではありません。

公開後

  1. Search Consoleへドメインを登録し、サイトマップを送信します。
  2. URL検査でトップページと主要記事が取得できるか確認します。
  3. クロールエラー、重複ページ、インデックス除外理由を確認します。
  4. 更新後はサイトマップの日付と記事の更新日を正しく変更します。
ポイント

技術SEOは土台です。設定だけではなく、検索意図を満たす本文と内部リンクが必要です。

掲載内容は一般的な情報です。実際の仕様、料金、制度、営業時間などは公式情報をご確認ください。
