Googleスプレッドシート

Googleスプレッドシートのピボットテーブルで月次集計を作る

明細データから店舗別・担当者別・月別の売上や件数を集計し、更新しやすいダッシュボードを作ります。

公開日: 2026-07-22最終更新: 2026-07-22
この記事の内容
  1. 元データの条件
  2. 月別売上の作成手順
  3. ダッシュボード構成
  4. よくあるずれ

元データの条件

ピボットテーブルは、1行が1件の明細で、各列に見出しがある縦長データと相性が良い機能です。結合セル、小計行、複数段見出しがある表は先に整形します。

日付、店舗、担当者、区分、金額など、集計に使う項目を明細へ持たせます。

月別売上の作成手順

  1. 明細範囲を選択してピボットテーブルを挿入します。
  2. 行に店舗または担当者を追加します。
  3. 列に日付を追加し、必要に応じて年月でグループ化します。
  4. 値に売上金額を追加し、集計方法を合計にします。
  5. 区分やステータスをフィルタへ追加します。

ダッシュボード構成

  • 売上合計と件数を別の値として表示。
  • 店舗、担当者、商品区分で切り替えられるようにする。
  • 元データ更新後の集計範囲を確認する。
  • グラフは意思決定に必要な2～3種類へ絞る。

よくあるずれ

  • 金額が文字列になっていて合計されない。
  • 日付が文字列のため月ごとにまとまらない。
  • 空白や表記揺れで同じ店舗が別項目になる。
  • 明細範囲に新しい行が含まれていない。
  • キャンセルやテストデータの除外条件がない。
ポイント

ピボットを作る前に、元データの型とマスタ表記を統一すると集計精度が上がります。

掲載内容は一般的な情報です。実際の仕様、料金、制度、営業時間などは公式情報をご確認ください。
