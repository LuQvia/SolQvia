GASのUrlFetchAppでJSON APIと連携する方法

GET・POST、HTTPヘッダー、JSON変換、エラー判定を含む外部API連携の基本を解説します。

外部API連携の流れ

UrlFetchAppを使うと、Apps Scriptから外部のHTTP APIへリクエストできます。URL、HTTPメソッド、ヘッダー、本文、タイムアウトやエラー処理を仕様書に合わせます。

APIキーやトークンはコードへ直書きせず、設定ストアへ分離します。ログにも全文を出力しないようにします。

POSTの例

function createItem(payload) {
  const token = PropertiesService.getScriptProperties().getProperty('API_TOKEN');
  if (!token) throw new Error('API_TOKEN is not configured');

  const response = UrlFetchApp.fetch('https://api.example.com/items', {
    method: 'post',
    contentType: 'application/json',
    headers: { Authorization: `Bearer ${token}` },
    payload: JSON.stringify(payload),
    muteHttpExceptions: true
  });

  const status = response.getResponseCode();
  const body = response.getContentText();
  if (status < 200 || status >= 300) {
    throw new Error(`API error ${status}: ${body.slice(0, 300)}`);
  }
  return JSON.parse(body);
}

確認すべき仕様

  • 認証方式とトークンの有効期限。
  • Content-Typeと文字コード。
  • 成功時と失敗時のHTTPステータス。
  • ページネーションの形式。
  • レート制限と再試行条件。
  • 同じリクエストを再送した場合の重複動作。

運用上の注意

  1. まず小さなテストデータで接続を確認します。
  2. レスポンスの必要項目だけを保存します。
  3. 429や500番台のみ、上限付きで再試行します。
  4. 処理件数が多い場合はキューとトリガーで分割します。
  5. API仕様変更に備え、バージョンと最終確認日を記録します。
ポイント
