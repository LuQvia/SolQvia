Google Apps Script

GASのメール送信上限に備えたキュー設計

メールを一括送信せず、送信待ち・送信済み・失敗を管理してApps Scriptの割り当て超過を防ぐ方法です。

  2. キューの列設計

送信上限を前提にする

Apps ScriptとGoogle Workspaceの各サービスには日次などの割り当てがあり、アカウント種別や仕様変更によって値が変わることがあります。大量送信を1回の処理へ詰め込まず、現在の残り送信可能数を確認します。

広告メールや不特定多数への配信ではなく、組織内通知や申請結果など正当な用途でも、宛先数と再送を管理する必要があります。

キューの列設計

  • message_id: 一意な送信ID。
  • to、subject、body: 送信内容。
  • status: pending、sent、failed。
  • attempts: 試行回数。
  • next_attempt_at: 次回試行日時。
  • sent_at、last_error: 実績と障害内容。

分割送信の例

function processMailQueue() {
  const remaining = MailApp.getRemainingDailyQuota();
  if (remaining <= 0) return;

  const sheet = SpreadsheetApp.getActive().getSheetByName('MailQueue');
  const values = sheet.getDataRange().getValues();
  let sent = 0;

  for (let i = 1; i < values.length && sent < Math.min(remaining, 50); i++) {
    if (values[i][4] !== 'pending') continue;
    MailApp.sendEmail(values[i][1], values[i][2], values[i][3]);
    sheet.getRange(i + 1, 5, 1, 2).setValues([['sent', new Date()]]);
    sent++;
  }
}

運用上の注意

  1. 宛先を重複排除し、1メッセージ1IDを付けます。
  2. 送信前に残り割り当てを確認します。
  3. 1回の最大処理数を決め、時間トリガーで続行します。
  4. 一時的な失敗だけを再試行し、恒久エラーは保留します。
  5. 個人情報を含む本文や宛先一覧の閲覧権限を制限します。
ポイント

送信成功後に状態更新が失敗するケースもあるため、重複送信を検知できるID設計が必要です。
