GASを高速化する一括読み込み・一括書き込みの基本

セルを1件ずつ処理せず、配列へまとめて読み書きすることでApps Scriptの実行時間を短縮する方法を解説します。

公開日: 2026-07-22最終更新: 2026-07-22

  2. 一括処理の手順

遅くなる典型的な原因

Apps Scriptでは、JavaScript内の計算よりもSpreadsheetサービスなど外部サービスへの呼び出しに時間がかかります。ループ内でgetValueとsetValueを繰り返す構成は、行数が増えるほど実行時間が伸びやすくなります。

基本方針は、必要な範囲をgetValuesで一度に配列へ読み込み、配列上で加工し、最後にsetValuesでまとめて戻すことです。

一括処理の手順

  1. 対象範囲の開始行、列、行数、列数を確定します。
  2. getValues()で2次元配列として読み込みます。
  3. mapやforループで配列の値を加工します。
  4. 出力配列の行数と列数が書き込み範囲と一致するか確認します。
  5. setValues()で一度に書き戻します。

コード例

function updateStatuses() {
  const sheet = SpreadsheetApp.getActive().getSheetByName('Tasks');
  const lastRow = sheet.getLastRow();
  if (lastRow < 2) return;

  const values = sheet.getRange(2, 1, lastRow - 1, 4).getValues();
  const output = values.map(row => {
    const due = row[2];
    const done = row[3] === true;
    const status = done ? '完了' : (due instanceof Date && due < new Date() ? '期限超過' : '進行中');
    return [status];
  });

  sheet.getRange(2, 5, output.length, 1).setValues(output);
}

設計上の注意

  • 空行を含む全列参照ではなく、実データ範囲を取得する。
  • 書き込み回数を減らすため、出力先が離れていても可能な範囲でまとめる。
  • 日付、真偽値、数値を文字列へ不用意に変換しない。
  • 大量データでは処理単位を分割し、次回開始位置を保存する。
ポイント

速度改善では、サービス呼び出し回数を数えると問題箇所を見つけやすくなります。
