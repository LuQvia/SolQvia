GASの時間主導トリガーで定期処理を自動化する方法

毎日・毎時・指定曜日に処理を動かす設定方法と、二重実行や失敗に備える設計を説明します。

時間主導トリガーとは

時間主導トリガーは、指定した間隔や時間帯にApps Script関数を自動実行する仕組みです。日次集計、未処理データの確認、定期通知、バックアップ作成などに使えます。

指定時刻ぴったりではなく、設定した時間帯の中で実行される場合があります。厳密な秒単位の実行を前提にしない設計が必要です。

画面から設定する方法

  1. Apps Scriptの左メニューからトリガーを開きます。
  2. トリガーを追加し、実行する関数を選択します。
  3. イベントのソースを「時間主導型」にします。
  4. 日付ベース、時間ベース、分ベースなどの間隔を選びます。
  5. 失敗通知の設定を確認し、保存します。

コードで作成する例

function createDailyTrigger() {
  deleteTriggers_('dailyJob');
  ScriptApp.newTrigger('dailyJob')
    .timeBased()
    .everyDays(1)
    .atHour(9)
    .create();
}

function deleteTriggers_(handlerName) {
  ScriptApp.getProjectTriggers()
    .filter(t => t.getHandlerFunction() === handlerName)
    .forEach(t => ScriptApp.deleteTrigger(t));
}

function dailyJob() {
  // Daily processing goes here.
}

安定運用の設計

  • トリガー作成前に同名トリガーを削除し、重複実行を防ぐ。
  • 処理済みIDや最終処理日時を保存し、再実行しても重複更新しないようにする。
  • LockServiceで同時実行を制御する。
  • 外部APIやメール送信の失敗をログへ残す。
  • 長時間処理は分割し、次回実行位置をプロパティに保存する。
ポイント

定期処理は『同じ処理を2回実行しても結果が壊れない』冪等性を意識すると保守しやすくなります。
