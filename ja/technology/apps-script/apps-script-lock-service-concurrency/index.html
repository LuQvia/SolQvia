GASの同時実行をLockServiceで制御する方法

フォーム送信やWebアプリの同時アクセスで連番や残高が重複しないよう、LockServiceで排他制御する方法を解説します。

  2. ロックの種類

同時実行で起きる問題

複数のフォーム送信やWebアプリ操作がほぼ同時に発生すると、双方の処理が同じ最終行や同じ連番を読み取り、重複した値を書き込むことがあります。

共有資源の読み取りから更新までを1つの排他的な区間として扱うことで、競合を減らせます。

ロックの種類

  • Script lock: スクリプト全体で同時に1処理だけ通す。
  • User lock: 同じユーザーの重複実行を抑える。
  • Document lock: 対象ドキュメント単位で競合を抑える。

連番採番の例

function issueNumber() {
  const lock = LockService.getScriptLock();
  lock.waitLock(10000);
  try {
    const props = PropertiesService.getScriptProperties();
    const current = Number(props.getProperty('NEXT_NUMBER') || 1);
    props.setProperty('NEXT_NUMBER', String(current + 1));
    return `REQ-${String(current).padStart(6, '0')}`;
  } finally {
    lock.releaseLock();
  }
}

安全な使い方

  • ロック取得後の処理を必要最小限にする。
  • waitLockの待機時間を設定し、永久待機を避ける。
  • try/finallyで必ずreleaseLockを呼び出す。
  • 外部API待ちや大量計算はロック区間の外へ出す。
  • ロックだけに頼らず、一意キーや重複チェックも組み合わせる。
ポイント

排他制御は競合を減らしますが、業務上の一意性を保証する検証も別途必要です。
