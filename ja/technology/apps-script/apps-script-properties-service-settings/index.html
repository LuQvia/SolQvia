Google Apps Script

GASのPropertiesServiceで設定値と処理状態を安全に管理する

スプレッドシートへ直接書きたくない設定値、最終処理位置、ユーザー別設定をPropertiesServiceで管理する方法です。

PropertiesServiceが向く情報

PropertiesServiceは、Apps Scriptプロジェクト内で文字列のキーと値を保存する仕組みです。APIの接続先、通知先ID、最終処理行、前回実行日時など、小さな設定値や状態管理に向いています。

大量データや機密性の高い資格情報を無制限に保存するデータベースではありません。保存上限とアクセス範囲を理解して使います。

3種類の使い分け

  • Script Properties: スクリプト全体で共有する設定。
  • User Properties: 実行ユーザーごとの表示設定や選択値。
  • Document Properties: 対象ドキュメントにひも付くアドオン向け設定。

コード例

function saveSettings() {
  const props = PropertiesService.getScriptProperties();
  props.setProperties({
    SLACK_CHANNEL_ID: 'C0123456789',
    LAST_ROW: '250',
    LAST_RUN_AT: new Date().toISOString()
  });
}

function loadSettings() {
  const props = PropertiesService.getScriptProperties().getProperties();
  const lastRow = Number(props.LAST_ROW || 1);
  return { ...props, lastRow };
}

運用ルール

  1. キー名を大文字の定数形式などに統一します。
  2. 数値やJSONは保存時に文字列化し、取得時に型変換します。
  3. 必須設定がない場合は処理開始前に明確なエラーを出します。
  4. 不要になったキーは削除し、古い設定を残さないようにします。
  5. 秘密情報は閲覧権限やログ出力にも注意し、必要に応じて専用の秘密管理基盤を検討します。
ポイント

設定値と業務データを分離すると、シート構成を変更してもスクリプトを保守しやすくなります。
