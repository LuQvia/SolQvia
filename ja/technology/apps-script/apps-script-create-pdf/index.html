GASでGoogleドキュメントからPDFを自動作成する方法

テンプレート文書を複製し、差し込み置換してPDF保存する実務向けの基本フローを解説します。

自動PDF作成の流れ

Apps Scriptでは、Googleドキュメントのテンプレートを複製し、本文中のプレースホルダーを置換してからPDFへ変換できます。見積書、通知書、申込内容の控えなど、同じレイアウトで内容だけが変わる書類に向きます。

テンプレートを直接編集せず、毎回コピーを作る構成にすると原本を壊しにくくなります。

準備するもの

  • GoogleドキュメントのテンプレートID
  • PDF保存先のGoogle DriveフォルダID
  • 置換する項目名と値
  • ファイル名の命名ルール

基本コード

function createPdf(data) {
  const templateId = 'DOCUMENT_TEMPLATE_ID';
  const folder = DriveApp.getFolderById('OUTPUT_FOLDER_ID');
  const copy = DriveApp.getFileById(templateId)
    .makeCopy(`document_${data.id}`, folder);

  const doc = DocumentApp.openById(copy.getId());
  const body = doc.getBody();
  body.replaceText('{{NAME}}', data.name);
  body.replaceText('{{DATE}}', data.date);
  body.replaceText('{{AMOUNT}}', String(data.amount));
  doc.saveAndClose();

  const pdfBlob = copy.getAs(MimeType.PDF)
    .setName(`document_${data.id}.pdf`);
  const pdfFile = folder.createFile(pdfBlob);
  copy.setTrashed(true);
  return pdfFile.getUrl();
}

品質確認のポイント

  • 置換文字列に正規表現で特別な意味を持つ記号を多用しない。
  • 長い氏名や住所でもレイアウトが崩れないか確認する。
  • 金額・日付はコード側で表示形式を統一する。
  • 一時コピーを削除する前にPDF作成が成功したか確認する。
  • 法定書類は、項目要件と保存要件を別途確認する。
ポイント

大量作成では、処理時間上限に備えて1回の件数を区切り、処理済みフラグを記録すると復旧しやすくなります。
