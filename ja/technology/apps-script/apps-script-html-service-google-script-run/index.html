Google Apps Script

GASのHTML Serviceで入力画面を作る基本

doGet、HTMLファイル、google.script.runを使い、スプレッドシート連携の入力画面を安全に作る方法です。

公開日: 2026-07-22最終更新: 2026-07-22
この記事の内容
  1. 画面とサーバー処理を分ける
  2. 最小構成

  4. 品質チェック

画面とサーバー処理を分ける

HTML Serviceでは、HTML、CSS、ブラウザ側JavaScriptを使った画面を作り、google.script.runでサーバー側のApps Script関数を非同期に呼び出せます。

画面側は入力と表示、サーバー側は権限が必要なデータ処理と検証を担当させます。ブラウザ側の検証だけを信用しないことが重要です。

最小構成

// Code.gs
function doGet() {
  return HtmlService.createHtmlOutputFromFile('Index')
    .setTitle('Request Form');
}

function saveRequest(data) {
  if (!data || !String(data.name || '').trim()) {
    throw new Error('Name is required');
  }
  const sheet = SpreadsheetApp.getActive().getSheetByName('Requests');
  sheet.appendRow([new Date(), String(data.name).trim(), String(data.note || '')]);
  return { ok: true };
}

クライアント側の呼び出し

document.querySelector('#form').addEventListener('submit', event => {
  event.preventDefault();
  const button = event.submitter;
  button.disabled = true;

  google.script.run
    .withSuccessHandler(() => {
      button.disabled = false;
      document.querySelector('#message').textContent = 'Saved';
    })
    .withFailureHandler(error => {
      button.disabled = false;
      document.querySelector('#message').textContent = error.message;
    })
    .saveRequest({ name: form.name.value, note: form.note.value });
});

品質チェック

  • 送信中はボタンを無効化し、二重送信を防ぐ。
  • 成功・失敗ハンドラを必ず設定する。
  • HTMLへ秘密情報やアクセストークンを埋め込まない。
  • 外部スクリプトとスタイルはHTTPSで読み込む。
  • キーボード操作、ラベル、エラーメッセージを確認する。
ポイント

HTML Serviceはiframeサンドボックス内で動作するため、通常のWebページと異なる制約があります。

掲載内容は一般的な情報です。実際の仕様、料金、制度、営業時間などは公式情報をご確認ください。
