GASのCacheServiceで外部取得と重い計算を減らす

頻繁に変わらないAPI結果やマスタデータを一時保存し、Apps Scriptの実行速度と安定性を改善します。

キャッシュが向くデータ

CacheServiceは、一定時間だけ再利用できる値を保存し、外部APIや重い計算の回数を減らすために使います。設定の正本や失ってはいけないデータを置く場所ではありません。

キャッシュは期限前でも消える可能性があるため、取得できなければ元データから再生成できる設計にします。

キャッシュの種類

  • Script cache: スクリプト利用者で共有する一時データ。
  • User cache: ユーザーごとに異なる一時データ。
  • Document cache: ドキュメント単位の一時データ。

API結果を保存する例

function getReferenceData() {
  const cache = CacheService.getScriptCache();
  const key = 'reference-data-v1';
  const cached = cache.get(key);
  if (cached) return JSON.parse(cached);

  const response = UrlFetchApp.fetch('https://api.example.com/reference');
  const data = JSON.parse(response.getContentText());
  cache.put(key, JSON.stringify(data), 900);
  return data;
}

function clearReferenceCache() {
  CacheService.getScriptCache().remove('reference-data-v1');
}

設計のポイント

  1. キャッシュキーにデータ種別とバージョンを含めます。
  2. 更新頻度に合わせて有効期限を決めます。
  3. データ更新処理の後に関連キャッシュを削除します。
  4. 大きなデータは要約し、必要部分だけ保存します。
  5. キャッシュミス時の元処理が正常に動くかテストします。
ポイント

高速化のための補助層として使い、キャッシュがなくても正しい結果を返せる構成にします。
