Google Apps Script

Googleフォームの回答をSlackへ自動通知する方法

Apps ScriptとSlackのIncoming Webhookを使い、フォーム回答をチャンネルへ通知する構成を説明します。

公開日: 2026-07-22最終更新: 2026-07-22
この記事の内容
  1. 必要な構成
  2. 設定手順
  3. 通知コード
  4. 失敗しにくい運用

必要な構成

フォーム回答をSlackへ送るには、Slack側のIncoming Webhook URLと、Google側のフォーム送信トリガーを組み合わせます。Apps ScriptではUrlFetchApp.fetch()を使ってWebhookへJSONをPOSTします。

Webhook URLは外部に漏れると第三者から投稿されるおそれがあります。コードへ直接書くより、スクリプトプロパティに保存する方が管理しやすくなります。

設定手順

  1. SlackでIncoming Webhookを有効にし、通知先チャンネルのWebhook URLを取得します。
  2. Apps Scriptのプロジェクト設定でスクリプトプロパティにSLACK_WEBHOOK_URLを登録します。
  3. フォーム送信イベントを受け取る関数を作成します。
  4. インストール型トリガーを設定し、テスト回答を送信します。

通知コード

function onFormSubmit(e) {
  const url = PropertiesService.getScriptProperties()
    .getProperty('SLACK_WEBHOOK_URL');
  if (!url) throw new Error('Webhook URL is not configured.');

  const text = Object.entries(e.namedValues)
    .map(([key, value]) => `*${key}*\n${value.join(', ')}`)
    .join('\n\n');

  const response = UrlFetchApp.fetch(url, {
    method: 'post',
    contentType: 'application/json',
    payload: JSON.stringify({ text }),
    muteHttpExceptions: true
  });

  if (response.getResponseCode() >= 300) {
    throw new Error(response.getContentText());
  }
}

失敗しにくい運用

  • Webhook URLをスプレッドシートの閲覧可能なセルへ置かない。
  • 通知失敗時にApps Scriptの実行ログとHTTPステータスを確認する。
  • 個人情報をSlackへ送る前にチャンネル参加者と保存方針を確認する。
  • 夜間通知を抑制する場合は、いったんキュー用シートへ保存し、時間主導トリガーでまとめて送る。
ポイント

Slackの表示を整えたい場合はBlock Kitを使えますが、最初は単純なtext通知から安定性を確認するのが安全です。
