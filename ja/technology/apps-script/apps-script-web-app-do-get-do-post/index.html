GAS WebアプリのdoGetとdoPostを正しく使い分ける

画面表示、クエリパラメータ、外部サービスからのPOST受信をGAS Webアプリで実装する基本を整理します。

doGetとdoPostの役割

WebアプリへのGETリクエストではdoGet、POSTリクエストではdoPostが実行されます。画面を表示する場合はHtmlOutput、JSONやテキストを返すAPI風の処理ではTextOutputを使います。

本番URLと開発テスト用URLの違い、実行ユーザー、アクセス権限の設定を理解してから公開します。

受信データの確認

  • e.parameter: 単一値として扱うクエリまたはフォーム値。
  • e.parameters: 同名パラメータを配列で取得。
  • e.postData.contents: JSONなどPOST本文の文字列。
  • e.pathInfo: URL末尾の追加パス情報。

JSON受付の例

function doPost(e) {
  try {
    const payload = JSON.parse(e.postData.contents || '{}');
    if (!payload.id) throw new Error('id is required');

    const result = { ok: true, receivedId: String(payload.id) };
    return ContentService.createTextOutput(JSON.stringify(result))
      .setMimeType(ContentService.MimeType.JSON);
  } catch (error) {
    return ContentService.createTextOutput(JSON.stringify({
      ok: false,
      error: error.message
    })).setMimeType(ContentService.MimeType.JSON);
  }
}

公開前チェック

  1. 誰の権限で実行するかを決めます。
  2. アクセス可能なユーザー範囲を最小限にします。
  3. 受信値を型・長さ・必須項目で検証します。
  4. 秘密トークンをURLのクエリへ不用意に含めません。
  5. スプレッドシート更新には重複防止とロックを実装します。
